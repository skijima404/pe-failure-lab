## Failure Mode Statement
The platform still requires heavy enabling to be usable, so users cannot convert adoption into independent repeated use without continued human guidance.

## Context
This tends to emerge when the platform can be made to work for users, but only through substantial enablement effort that has not yet been converted into a simpler usage path.

## Mechanism
- Users need repeated explanation, coaching, or mediation before they can complete common usage successfully.
- The platform depends on human translation and support instead of making the intended path self-evident enough to use alone.
- As a result, usage may happen, but it does not become self-service behavior.

## Countermove
- Reduce the amount of human interpretation needed for common usage paths.
- Ask what a user can do independently after enablement ends.
- Treat repeated need for the same enabling support as a self-service design failure, not only as onboarding work.
