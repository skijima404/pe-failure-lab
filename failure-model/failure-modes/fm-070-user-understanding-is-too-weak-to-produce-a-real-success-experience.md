## Failure Mode Statement
User understanding is too weak to produce a real success experience, so the delivered output reflects assumptions that do not match how users actually work, understand, or adopt.

## Context
This tends to emerge when persona definition stays shallow, user literacy is assumed incorrectly, or the team does not validate what users can realistically recognize as success.

## Mechanism
- The team assumes a level of user readiness, literacy, or behavior that does not hold in practice.
- Persona definitions remain too broad or too idealized to guide design toward an actual successful experience.
- As a result, the output exists, but users do not experience the promised gain in a way that feels real or usable.

## Countermove
- Validate what the target users actually understand, need, and can adopt.
- Treat mismatch between assumed and actual user literacy as a win-design failure, not only as an adoption issue.
- Ask what concrete success experience the user should be able to describe after using the output.
