## Failure Mode Statement
Improvement loops run too intermittently to create acceleration, so the effort learns and adapts in occasional bursts rather than through a dependable rhythm.

## Context
This tends to emerge when loop-like activities exist in theory, but happen too irregularly, too weakly, or too reactively to sustain continuing change.

## Mechanism
- Feedback, review, or improvement activity occurs only when someone remembers, pushes, or feels enough pain.
- The loop is present episodically, but not consistently enough to shape ongoing behavior.
- As a result, improvement cannot compound because the effort never develops a stable learning cadence.

## Countermove
- Check whether improvement loops run consistently enough to influence real operating behavior over time.
- Replace pain-triggered or memory-triggered loops with a more dependable cadence.
- Treat intermittent looping as a structural weakness, not as a sign that a loop exists.
