## Failure Mode Statement
The win is designed for the wrong user persona, so the delivered output does not create recognizable success for the people whose adoption or behavior actually matters.

## Context
This tends to emerge when the effort assumes a user such as developers directly, even though the real leverage point sits with a different internal persona such as project managers, delivery leads, or other mediating roles.

## Mechanism
- The team defines success around the wrong adoption surface.
- Delivered outputs are optimized for a persona that is not the real gate, influencer, or day-to-day user of the change.
- As a result, something is built, but it does not land as meaningful success where organizational behavior would actually shift.

## Countermove
- Recheck who must actually succeed for the short-term win to count as progress.
- Treat persona choice as a win-design decision, not only as a discovery artifact.
- Ask whether the delivered result would be experienced as success by the real adoption path, not only by the intended technical audience.
